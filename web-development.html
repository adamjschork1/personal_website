Code experience

HTML & CSS

I've lumped HTML and CSS together here becasue these aren't coding languages and also because knowing HTML is not an impressive skill. CSS on the other hand is the part that takes a lot of hand on experience. This is becasue the number of possible ways a web page can be styled are nearly limitless, and while HTML has not changed a lot over the years, CSS has actually changed quite a bit (and still is). Having a fairly good comprehension of how to style a page is a must, but as one dives into all of the fancy and newer features, it's important to code for compatability for multiple browsers and devices.

My approach to building a page is fluid, strategic and proactive. My goal is to write in the most efficeint way possible to reduce going back to prior code to clean it up or modify it. Whether I start building out a mobile version first or for desktop, I always keep in mind how one will look compared to the other, where to make breakpoints and how many to make. With so many devices and screen sizes these days, I don't follow a rigid screen breakpoint rule. My approach is not to design for specific devices, but to rearrange a page's content and design where it makes sence, organicly.

I have familiarity with various CSS libraries, including Bootstrap, Pure and SASS (although SASS is actually a scripting language). When it comes to a new library, I can research its naming conventions and adapt fairly quickly. I generally try to avoid importing and using libraries unless there are project time restrictions or preexisting use, in which case keeping the website cohesive neccesitates it. I have some experience with jquery, though I try to avoid jquery dependency.

Overall, trying to be as objective as possible, I give myself a three out of five stars in this category. I created this rating comparing myself to a spectrum of skill level, starting from zero knowledge and going up to a ten year seasoned pro. I've been doing this for several years, and based on my expereince, I am comfortable and confident in this category.

Liquid

Liquid is a language created by and used on Shopify. For those not in ecomm, Shopify is a widely used e-comm platform for online retailers. Shopify is built with the Ruby on Rails framework. Shopify's github describes Liquid as such: "Liquid is an open-source template language created by Shopify and written in Ruby. It is the backbone of Shopify themes and is used to load dynamic content on storefronts."

Liquid is very easy to use, similar to HTML, but does have its own unique features. Since it is a template language, learning it is not difficult at all. I've been working with Liquid for over a year now, and haven't made any mistakes with it yet. This includes dynamically loading content and writing conditional functions. I give myself a three out of five. With another year of experience I will probably add another star to my self rating.
